# FundTogether 众筹平台 (Vibe Coding Project)

## 项目简介

FundTogether 是一款全功能在线众筹平台，支持多端自适应响应。系统旨在为项目发起人、普通支持者及平台管理员提供一站式的众筹服务闭环。包含用户权限管理、项目创建审核、支付交易流水、数据报表可视化以及多端实时消息通知等核心模块

## 👥 角色与权限系统

平台包含以下核心角色：

1. **游客**：浏览项目、注册账号。
2. **普通支持者**：登录系统、完善资料、绑定支付、支持项目、留言互动、管理个人订单与动态。
3. **项目发起人**：创建众筹项目、管理待审核/筹款中项目、发布项目动态、设置回报档位、回复留言、查看/导出数据等。
4. **系统管理员**：分配用户权限、审核上线项目、监管下架违规内容、发布全站公告、查看及导出核心运营数据。

## 🛠️ 核心功能模块

### 1. 认证与用户管理 (Authentication & User Profile)

- **注册与登录**：支持手机号/邮箱注册及登录，生成专属账号并分配默认权限（游客注册、注册用户登录、找回登录密码）。
- **账号安全与设置**：支持登录用户安全退出、修改登录密码。
- **个人资料维护**：查看个人完整资料、修改基础信息（昵称/简介等）、上传/更换头像。
- **实名与支付绑定**：未实名用户上传实名认证资料、普通支持者绑定多渠道支付方式（微信/支付宝/银行卡）。

### 2. 项目浏览与交互 (Project Browsing & Interaction)

- **项目大厅**：查看全量已上线众筹项目，支持按热度排序；首页展示热门项目排行榜。
- **多维度筛选与搜索**：支持关键词精准搜索，及按分类、金额、进度、热度进行多维度/二次筛选。
- **个性化推荐**：依据用户兴趣、支持与浏览历史，个性化推荐相关项目。
- **详情展示**：展示项目完整信息、视频介绍，以及直观的每日筹款趋势折线图。
- **互动社区**：登录用户可对项目及他人评论进行点赞，发布项目评论，发起人可回复留言。

### 3. 项目支持与交易闭环 (Support & Transactions)

- **支持项目**：登录用户可选择不同金额档位及对应实物/权益回报提交支持，支持过程中可给发起人留言。
- **多渠道支付**：支持绑定微信、支付宝、银行卡等主流渠道完成支付，更新订单状态。
- **个人支持管理**：支持者可查看个人支持历史记录、累计金额、项目数统计及订单详情。

### 4. 项目发起与管理 (Project Creation & Management)

- **项目创建与审核流程**：发起人填写信息、上传资料/视频、设置支持档位与回报并提交审核；支持修改或取消待审核项目。
- **上线项目管理**：支持修改筹款中项目的非核心信息（描述、补充图片等），管理用户留言评论（删除违规内容）。
- **动态与反馈**：发起人可发布项目进度动态（同步支持者），回复用户评论。
- **数据追踪**：查看筹款核心数据（总额/进度/剩余天数）、项目访问与分享趋势数据。
- **支持者管理**：查看支持者名单、对应支持金额与留言，支持将名单导出为 Excel 文件。

### 5. 后台监管与运营 (Admin Dashboard)

- **用户监管**：查看全量用户账号列表，管理员可禁用违规账号，分配或修改用户角色权限（支持者/发起人/管理员）。
- **项目审核与风控**：查看待审核列表并进行通过/驳回操作，对已上线违规项目进行强制下架处理。
- **全平台监管**：查看平台所有项目状态进度与全量数据。
- **数据报表可视化**：查看核心运营数据、分类筹款趋势，通过多维度图表（柱状图、折线图、饼图）直观展示，并支持导出运营统计报表。
- **公告系统**：管理员发布平台公告、活动政策通知，可定向或全量推送至用户。

### 6. 消息与通知机制 (Messaging & Notifications)

- **系统通知**：支持者接收项目进展通知、动态更新；全站接收平台公告与系统政策推送。

### 7. 跨端适配与体验 (Cross-Platform Experience)

- **移动端响应式**：全站页面支持移动端浏览器响应式适配，无布局错乱。
- **移动端管理**：项目发起人可在手机端完成发布、编辑、管理等核心操作。
- **实时消息推送**：移动端支持 WebSocket 实时接收项目动态、公告、回复等提醒。

### 8. 约束与开发规范 (Constraints & Conventions)

> *注：本项目开发需严格遵循现有的代码及架构约束标准。*

- **代码规范**：确保前后端分离架构的接口对接清晰，遵循 RESTful API 设计风格（或现有约定）。
- **权限边界**：严格落实角色鉴权，任何写操作及敏感数据读取均需校验用户Token及对应角色权限。
- **状态流转**：订单状态（待支付、已支付）与项目状态（待审核、筹款中、已完成、已驳回、已取消、已下架）必须保证强一致性与事务安全。
- **支付与资金**：接入第三方支付渠道需满足合规性验证与异步回调机制安全。
- **文件存储**：图片、视频及导出文件的上传下载需对接标准云存储/CDN方案，确保大小与格式校验有效。
